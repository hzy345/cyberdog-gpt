# CyberDog GPT 对话系统

基于 GPT 的小米 CyberDog 机器狗智能对话系统，让机器狗能够理解语音、生成个性化回复、执行动作并播放语音。

## 项目简介

本项目实现了机器狗的完整对话交互流程：
1. **语音识别** - 接收用户语音输入
2. **智能回复** - 调用 GPT API 生成带有情感和动作的回复
3. **动作执行** - 根据回复内容执行相应动作（站立、坐下、转圈等）
4. **语音合成** - 将文本回复转换为语音播放
5. **情感表达** - 通过 LED 灯显示不同情感状态

## 文件结构

```

├── TTS_HTTP.py           # 语音合成 - 字节跳动 TTS API

├── Prompts/              # 提示词模板
│   ├── Marvin/           # 主角色提示词
│   └── Gilbert/          # 备用角色提示词
└── else_file/            # 早期版本文件（可忽略）
```

## 各模块功能

| 模块 | 功能 | 关键类/函数 |
|------|------|------------|
| `interactionNode.py` | ROS2 主节点，处理语音识别、LED 控制、动作执行 | `MyNode`, `asr_callback()` |
| `cyberdog_gpt.py` | GPT 对话核心，管理提示词和对话历史 | `PromptGenerator`, `sendReq()`, `interact()` |
| `TTS_HTTP.py` | 语音合成，调用字节跳动 API 生成语音 | `convert()`, `utilFunc()` |
| `intNodeInterface.py` | 定义交互接口抽象类 | `interactionNodeInterface`, `CyberdogResponse` |
| `Utils.py` | 工具函数 | `find_key_by_value()` |

## 实机运行

### 前置要求

1. Python 3.6+
2. ROS2 环境
3. 安装依赖：
   ```bash
   pip install requests
   ```

### 配置 API 密钥

在项目根目录创建 `api_keys.txt` 文件，每行一个 API 密钥：
```
sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
sk-yyyyyyyyyyyyyyyyyyyyyyyyyyyyyyyy
```

### 启动系统

在 CyberDog 机器狗上运行：
```bash
python3 interactionNode.py
```

系统启动后会自动：
1. 初始化 GPT 提示词
2. 订阅语音识别话题
3. 等待用户语音输入
4. 生成回复并执行动作/播放语音

## 支持的动作指令

| 动作 | 指令 ID | 说明 |
|------|---------|------|
| 站立 | 111 | 站起来 |
| 坐下 | 143 | 坐下 |
| 趴下 | 101 | 趴下 |
| 作揖 | 123 | 可怜地作揖 |
| 转圈 | 901 | 原地转圈 |
| 走正方形 | 902 | 走正方形路径 |
| 翻滚 | 903 | 在地上翻滚 |
| 扭屁股 | 144 | 扭屁股 |

## 情感状态

支持 21 种情感状态，包括：happy、sad、angry、scare、hate、surprise、pleased、sorry、annoyed 等。

## 注意事项

- 本项目需要在小米 CyberDog 机器狗的 ROS2 环境中运行
- API 密钥请妥善保管，不要提交到公开仓库
- 默认使用 `Prompts/Marvin` 作为角色提示词，可在代码中切换
